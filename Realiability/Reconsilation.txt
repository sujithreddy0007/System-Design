Reconciliation is the process of comparing the actual state of a system with the expected state and fixing any inconsistencies.

Example

Imagine a payment system:

User pays ₹1000.
Payment service marks payment as successful.
Due to a failure, the Order service never receives the success event.
Now:
Payment DB → Paid ✅
Order DB → Unpaid ❌

A reconciliation job runs later:

Finds the mismatch.
Updates the Order service.
Both systems become consistent.
Why Use It?

Distributed systems can experience:

Network failures
Service crashes
Lost messages
Partial updates

Reconciliation helps detect and repair these issues.

Real-world Examples
Payment and Order systems
