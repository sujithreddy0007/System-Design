Rate Limiting is a mechanism that restricts how many requests a user, client, or service can make within a specific time period.

Example

Suppose an API allows:

100 requests / minute

If a user sends:

Request 1 ✓
Request 2 ✓
...
Request 100 ✓
Request 101 ✗ (429 Too Many Requests)

The extra requests are rejected or delayed.

Why Use It?
Prevent abuse
Prevent DDoS attacks
Protect backend services
Ensure fair resource usage
Control costs
Common Algorithms
Fixed Window
100 requests/minute

Simple but can cause bursts at window boundaries.

Sliding Window

Tracks requests continuously over time.

More accurate but slightly expensive.

Token Bucket
Bucket Capacity = 100
Refill Rate = 10/sec

Allows short bursts while maintaining limits.

Leaky Bucket

Processes requests at a constant rate.

Smooths traffic spikes.
